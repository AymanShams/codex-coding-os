---
name: quant-review
description: Use when the user asks to calculate, estimate, forecast, model, sanity-check numbers, review a spreadsheet-like model, check assumptions, compare numeric options, validate a statistic, inspect a KPI, build or review a business case, test sensitivity, or explain the math behind a claim. Use for pricing models, financial estimates, forecasts, unit economics, conversion rates, margins, capacity, runway, ROI, market-share math, percentages, ratios, charts, tables, and numeric claims where formulas, inputs, units, assumptions, outputs, or sensitivity matter. Do not use for broad strategy without meaningful numbers, pure source verification, or casual opinions. If the primary task is pricing design or willingness-to-pay strategy, use pricing-strategy and add quant-review only for the numeric model. If the primary task is TAM, SAM, SOM, or addressable-market sizing, use market-sizing and add quant-review for calculations. If the primary task is a bootstrapped-startup CFO decision such as hiring, runway, burn, or capital allocation, use charlie and add quant-review when exact formulas or sensitivity are needed. If the primary task is source quality or citation verification, use evidence-checker. If the primary task is skeptical review of a broader argument or recommendation, use deep-critic.
---

For any calculation, estimate, pricing model, forecast, or statistic, always show:
- formula
- inputs
- units
- assumptions
- final result

If any input is missing, state the assumption and show the sensitivity if material.

Review numeric logic skeptically. Check whether the units are consistent, whether the assumptions are realistic, whether the formula actually matches the claim, and whether the conclusion is robust to reasonable changes in inputs.

## Required output sequence
1. Direct answer
2. Formula
3. Inputs and units
4. Assumptions
5. Calculation or model result
6. Sensitivity, if material
7. Risks or failure modes

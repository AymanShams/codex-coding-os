name: grill-me
description: "Use when the user asks to be grilled, interviewed, questioned, challenged through questions, interrogated on a plan, walked through a decision tree, or forced to clarify a plan, design, strategy, product idea, operating model, architecture idea, or decision before execution. Use when the user wants one-question-at-a-time pressure, shared understanding, branch-by-branch decision resolution, or recommended answers after each question. Use for prompts like grill me, interview me on this, ask me the hard questions, help me think this through question by question, or force me to clarify this plan. Do not use for normal critique without an interview format, failure-mode premortems, source verification, drafting deliverables, or codebase-specific terminology alignment. If the primary task is failure-first risk analysis, use pre-mortem. If the primary task is skeptical critique of an artifact or recommendation, use deep-critic. If the primary task requires checking existing project terminology, CONTEXT.md, ADRs, or code while grilling, use grill-with-docs."
---

Interview me relentlessly about every aspect of this plan until we reach a shared understanding. Walk down each branch of the design tree, resolving dependencies between decisions one-by-one. For each question, provide your recommended answer.

Ask the questions one at a time.

If a question can be answered by exploring the codebase, explore the codebase instead.

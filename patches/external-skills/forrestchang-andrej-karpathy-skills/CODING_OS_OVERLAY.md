# Coding OS Overlay For Karpathy-Inspired Coding Rules

Use this as a companion note beside the original external skill. Do not edit upstream files directly unless you are maintaining a fork.

## Attribution

Treat the four common rules as Karpathy-inspired, not verified verbatim Karpathy text.

## Local Adaptation

This pack expands the rules into a Codex workflow:

1. Think before coding.
2. Simplicity first.
3. Surgical changes.
4. Goal-driven execution.
5. Deterministic logic belongs in code.
6. Use phase limits and checkpoints.
7. Surface conflicts instead of blending conventions.
8. Read target files, imports, callers, and tests before editing.
9. Tests must prove behavior, not just existence.
10. Long work needs checkpoints and diff review.
11. Convention beats novelty.
12. Fail visibly.

## Codex-Specific Policy

- Use AGENTS.md for persistent repo rules.
- Use skills for reusable workflows.
- Use git status and git diff as review anchors.
- Do not pass secrets or production data to external agents.
- Use external skill packs only as references unless their license and fit are reviewed.
